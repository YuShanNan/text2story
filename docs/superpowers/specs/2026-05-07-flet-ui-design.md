# Flet 桌面 GUI 设计规格

## 概述

用 Flet 框架为 text2story 项目构建桌面 GUI，替代 CLI 命令，覆盖全部功能模块。

## 范围

全功能桌面 GUI：覆盖 SRT 修正、分镜生成、提示词生成、画面优化、视频提示词、流水线（阶段一+阶段二批量处理）。

## 流水线 Tab 详细设计

### 规则下拉

四个下拉框，同行从左到右排列，各自对应不同的提示词目录：

| 标签 | 对应目录 |
|------|----------|
| SRT修正 | `prompts/srt_correction/` |
| 分镜生成 | `prompts/storyboard/` |
| 画面优化 | `prompts/image_prompt_optimize/` |
| 视频提示词 | `prompts/video_prompt_from_image/` |

默认值均为 `default`，下拉内容自动读取对应目录下的 `.txt` 文件（不含扩展名）。

### 顺序/并行切换

- 默认：**顺序**。文件逐个串行跑完阶段一+阶段二
- 并行：多文件同时启动独立进程，需足够 API 额度
- 切换即生效，按钮文字联动变化：

| 模式 | 按钮文字 |
|------|----------|
| 顺序 | ▶ 顺序执行 |
| 并行 | ▶ 批量执行 |

### 批量执行逻辑（顺序模式）

```
文件1: SRT修正 → 分镜生成 → 完成
文件2: SRT修正 → 分镜生成 → 进行中
文件3: 等待
```

- 阶段一完成后暂停，等待用户检查分镜质量 → 确认后触发阶段二
- 可选"全自动"模式：阶段一 → 阶段二 连续执行
- 单文件失败标记 ✗ 并跳过，继续队列中后续文件

### 并行模式

- 所有选中文件同时启动独立 Python 进程
- 每个进程独立跑阶段一 → 阶段二
- 全部并行跑阶段一，完成后批量跑阶段二

## 其他 Tab 设计

单个功能 Tab（SRT修正/分镜生成/提示词生成/画面优化/视频提示词）均采用相同布局模式：

- 左侧：文件选择面板（单选）
- 右侧上方：配置栏（对应规则下拉+参数+执行按钮）
- 右侧下方：输出预览区
- 底部：进度条+日志

## 异步策略

- 执行按钮触发 `asyncio.create_task`，在后台线程运行现有 `core/` 模块的 generator 类
- 通过 `flet.Page.update()` 推送进度事件
- UI 不冻结，用户可切换 Tab 查看其他内容

## API 配置

- 启动时自动 ping API，成功显示 ✓ API已连接
- 失败弹出配置对话框，引导输入 API Key/URL/模型名
- 配置写入 `.env` 文件，与 CLI 共享

## 文件管理

- 左侧文件面板自动扫描 `input/` 目录下的文件
- 支持手动浏览选择任意路径
- 多文件模式支持复选框多选
- 输出自动保存到 `output/{stem}/` 目录
